1、JS错误信息类型

JavaScript有六种错误类型，每种错误类型都有构造函数，可以通过对错误类型的构造函数进行实例化进行人为的抛出错误。除了以下六种错误类型的构造函数外，还有Error构造函数。

1、SyntaxError 语法错误

变量名不规范：var 1 = 1 关键字赋值： new = 1 基本语法错误： var a = 1:

2、ReferenceError 引用错误

给无法赋值的对象赋值时：var a = 1 = 2；

3、RangeError 范围错误

数组长度赋值为负数：var arr = [1, 2]; arr.length = -1; 对象方法参数超出可行范围：var num = new NumberJ(66.66)；console.log(num.toFixed(-2));

4、 TypeError 类型错误

函数名类型错误： 123（） 函数形式调用对象属性：var obj = {}; obj.say(); 实例化初始值：var a = new 'string'；

5、URIError URI错误

错误的URI：var str = decodeURI('%fagagh%')

6、EvalError eval函数执行错误

2、try...catch...

try...catch...可以捕获错误并在catch中作出发生错误后的响应。 try{ console.log(a)//此处发生引用错误，不会再往下执行，跳转到catch块 console.log(b)//此处不执行 }catch(e){ console.log(e.name + ':' + e.message); }finally{ console.log('我一定执行');//此处无论try块中是否出现错误都会执行 }

3、严格模式

在js文件头部或块级作用域添加'use strict'，会给其下空间开启严格模式。 在严格模式下，变量必须声明，各函数内的this为undefined，函数的参数不能重复，对象的属性名不能重复（不报错），eval（）函数拥有自己的作用域。 例： 'use strict' a = 1;//报错 function test () { console.log(this); } test();//输出undefined function test2 (a, a) {}//报错 var obj = { a: 1, a: 2 }//严格模式不认同该种写法，但不报错 eval('var a = 1; ') console.log(a);//报错，eval中的a在自己的作用域中，而不是在全局
